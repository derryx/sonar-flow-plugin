The property "Pipeline Debug" should be set to "None" in production code.

If the property "Pipeline Debug" is set to "Save", "Restore (Override)" or "Restore (Merge)" this can lead to either performance problems or unexpected behaviour of a flow service.

Noncompliant Code Example

  The property "Pipeline Debug" is set to something else than "None".

Compliant Solution

  Set the property "Pipeline Debug" to "None".
